sql structure

RuffCo_: the database name

  tables : functions
    - columnNames
 
  planes: stores the planes and their attributes
    includes
    - plane_id
    - name varchar
    - model varchar
    - capacity int
    - mile_range int
    - current_location varchar
    - cruise_speed int

  reservations: link employees, planes, guests, and locations to a trip booked
    includes
    - flight_id 
    - date dateTime
    - plane_id (from planes)
    - dest_id (from destinations)
    - employee_id (from employees)
    - guests (Guest's ids are stores as a varchar, separeated by commas , )

  destinations: preset or added destinations employees can choose
    includes
    - dest_id
    - location varchar
    - distance_from_littlerock int
  
  employees:Employees registered to reserve jets
    includes
    - employee_id
    - first_name varchar
    - last_name varchar
    - email varchar
    - phone real
    - username varchar
    - password - encryption?

  guests: Guests that employees have registered
    includes
    - guest_id
    - first_name varchar
    - last_name varchar
    - email varchar
    - phone real
    - employee_id (This is how someone would be linked to the affiliated employee)
